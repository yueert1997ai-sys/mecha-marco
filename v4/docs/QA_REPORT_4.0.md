## 4.1 全机体重制验证

4.1 的正式机体层已从 Canvas 色块角色切换为 Three.js WebGL 3D：

- 页面加载固定版本 `three@0.149.0`
- `mech-3d-canvas` 独立叠加在 2D 场景之上
- WebGL 启动成功时，旧 Canvas 机体自动关闭；Three.js 或 WebGL 失败时恢复本地 Canvas 兜底
- 三台玩家机体具备独立比例、轮廓、装甲、背包和武器结构
- 敌方近战、狙击、炮击、坦克、精英和首领具备不同 3D 结构
- 模型使用斜面厚度装甲、金属材质、方向光、深色边缘线、接触阴影和三分之四俯视姿态
- 移动步态、射击后坐、军刀展开、推进焰和受击闪白由实际战斗状态驱动
- WebGL 层独立绘制敌人血条，避免切换 3D 后丢失原有战斗信息
- Three.js 依赖未写入离线缓存；本地 3D 启动代码、调参层和 Canvas 兜底均进入 Service Worker 缓存

## 4.1 真实浏览器截图测试

使用 GitHub Actions 中的 Chrome/Playwright 运行真实页面，而非只检查 UA 或字符串：

- 956×440 横屏、DPR 3、移动端模式、触控开启、iPhone Safari User-Agent
- 要求 `data-mech3d="ready"`、WebGL Context 存在且模型版本为 `4.1.0-webgl-hard-surface`
- 分别选择先锋型、重装型、高机动型
- 每台机体都点击开始出击、进入路线页、选择战斗房并等待实际渲染
- 三台机体均进入 `combat` 状态，WebGL 状态保持 `ready`
- 保存基地截图、三台机体战斗截图和 844×390 紧凑横屏截图
- 页面运行期间没有 JavaScript `pageerror`
- 最终截图用于修正第一版模型过大、正面直立和过曝问题；修正后采用缩小战斗尺度、三分之四俯视姿态、较低曝光、深色面板线与接触阴影

最终 WebGL QA 与系统测试均通过。该验证确认了浏览器中的真实模块加载、WebGL 创建、3D 模型生成、点击流程和截图输出。

## 已知边界

- 自动化环境使用 Chromium 与 SwiftShader，不能等同于 iPhone 17 Pro Max 的 Safari/WebKit GPU 驱动
- 尚未完成实体 iPhone 17 Pro Max 的 30 分钟发热、功耗、内存与持续帧率测试
- Three.js 当前从固定版本 CDN 加载；网络失败时会使用 Canvas 兜底，后续可改为仓库内自托管
- 当前机体为程序化原创硬表面模型，尚未替换成美术软件导出的正式 GLB 资产
- 仍需补充更完整的区域美术、实体障碍物导航、手柄输入和正式音频许可清单
