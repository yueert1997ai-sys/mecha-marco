## 合并后真实浏览器交互验证

在 PR #8 合并到 `main` 后，从 GitHub Actions 导出实际 `v4/` 运行目录，并在 Chromium 144 中通过 CDP 执行完整交互，不再只检查 UA 或页面字符串。

实际覆盖：

- 956×440 横屏视口，DPR 3，粗指针与 5 点触控，安全区模拟为左右 47px、底部 21px
- 使用真实命中坐标选择重装机体，进入路线页并选择战斗房
- 同时按住左右摇杆：玩家横向移动，右摇杆进入自动射击保持态
- 触控军刀、导弹与推进按钮：军刀产生斩击，导弹生成 2 枚，推进计时进入激活状态
- 440×956 竖屏切换后，横屏提示覆盖完整视口
- 844×390 较小横屏下，三张机体卡、开始出击按钮和面板全部保持在视口内
- 956×440 与 844×390 均无页面滚动、面板滚动或底部裁切
- 单帧 Canvas 渲染耗时约 3.1ms（当前无持续压力结论，仅用于确认渲染函数能实际执行）

真实交互测试发现并修复了两项此前静态测试未发现的回归：

1. `base-panel` 同时被加在 Overlay 和 Panel 上，裸 `.base-panel` 选择器误改外层安全区 Padding，造成基地面板左右不对称；已限定为 `.panel.base-panel`。
2. 右摇杆与推进/军刀按钮存在命中区域重叠，暂停键与资源栏也有视觉重叠；已重新排布动作按钮并为资源栏预留暂停键空间。

该验证属于真实 Chromium 布局、Canvas、点击与多点触控事件测试。它仍不能代替实体 iPhone 17 Pro Max 的 Safari 合成链路、触觉、温度、功耗和 30 分钟持续运行测试。
