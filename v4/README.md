# Mecha Marco / 天穹断刃 4.2.1

面向 iPhone Safari 与桌面浏览器的纯俯视机甲动作肉鸽纵切。当前版本为 `4.2.1-mobile-frontline-polish`：保留上半身俯视机体轮廓和十二段连续战线，重点完成横屏一屏适配、统一触控布局、设施目标、实体障碍和场景身份深化。

## 当前纵切

- 三台差异化机体
- 主射击、军刀、推进、重火力、超限五个动作槽位
- 30 个基础/稀有模块与 6 个组合协议
- 12 段连续轨道墓场战线，普通段清场后穿过闸门推进
- 普通战、精英战、商店、维修、事件与三阶段区域首领
- 敌人预警、受击硬直、方向性击退、受击停顿和阶段状态机
- 基地、战役报告、永久资源与条件对话
- 键鼠和 iPhone 横屏左摇杆、右侧主炮拖曳瞄准/射击、分层技能按钮操作
- iPhone 17 Pro Max 级横屏动态视觉视口、安全区、旋转恢复与无页面滚动适配
- Three.js 0.149.0 WebGL 显示层；网络或 WebGL 不可用时自动回退到本地 Canvas 版本

## 4.1 全机体视觉重制

- 正式战斗画面改为真实 WebGL 3D 机体，不再使用旧版线段、圆形和纯色多边形作为主角色
- 使用带斜面厚度的 `ExtrudeGeometry` 装甲、金属材质、方向光、边缘线和接触阴影
- 先锋型重做为锐利、均衡的泛用突击机，具备独立胸甲、肩甲、腰甲、四肢、背包、步枪和军刀
- 重装型重做为宽肩、厚胸、重腿、双反应炉背包和大型重武器的堡垒机
- 高机动型重做为窄躯干、轻装甲、展开式翼板和多推进节点的截击机
- 敌方普通、近战、狙击、炮击、坦克、精英与首领同步使用不同结构和比例的 3D 装甲机体
- 三台机体采用白/灰/深色骨架为主，识别色只用于胸部、传感器、推进器和武器细节
- 新模型继续读取战斗状态：移动步态、射击后坐、军刀展开、推进焰和受击闪白实时联动
- 3D 层独立恢复敌人血条，并为机体增加接触阴影和三分之四俯视姿态
- 基地选择卡改为三台新机体的专属预览，不再显示通用渐变徽章
- Canvas 投影装甲版本仅作为 Three.js 加载失败或 WebGL 不可用时的本地兜底

## 4.1.1 手机手感、镜头与性能

> 本节保留历史变更记录；当前移动端方案以 4.2.1 的统一布局与 `src/ui/mobileViewport42.js` 为准。

- 双摇杆采用独立死区与响应曲线，右摇杆轻推即可进入瞄准，超过阈值连续开火
- 触屏转向、机体朝向、移动起步和收杆响应加快，受击后移动不再严重发黏
- 推进期间允许继续主射击，移动、瞄准、开火与推进可以同时输入
- 镜头保持战斗场地居中，只提供有限的瞄准方向前视，冲刺不会把场地甩出屏幕
- 玩家出生点向场内收拢，完整机体和脚部保持在战斗区域内
- 玩家机体增加移动侧倾、冲刺俯身、头部补偿、持枪后坐和军刀手臂联动
- 自动目标提示环帮助手机端确认主要攻击目标
- 普通敌人使用低 draw-call 3D 骨架，保留头、胸、腰胯、手臂、腿和武器轮廓；精英、Boss 与玩家保持完整硬表面模型
- 手机 3D 层使用较低内部像素比，关键装甲保留边缘线，次要部件减少独立描边
- 956×440、DPR 3、双指触控与 SwiftShader QA 中，平均帧间隔由约 67ms 降至约 39ms

## 4.0.1–4.0.2 移动端修正

- 基地界面在短高度横屏下保持三台机体完整可见，底部按钮避开 Home Indicator
- 战斗视野扩大，玩家与敌人相对画面尺寸降低
- 房间布局增加专属地面标识和环境光差异
- 光束、导弹、军刀加入更清晰的核心、拖尾和分层刀光
- 敌我命中加入短促 hit-stop、镜头震动和冲击环
- 基地面板安全区、右摇杆动作按钮与暂停键重叠问题已通过真实 Chromium 交互测试修复

## 架构

```text
src/
  core/      固定步长循环、数学、事件总线、状态机
  combat/    Hardpoint Rig、技能缓冲、命中、移动手感与反馈打磨
  actors/    玩家机体与敌人状态机
  run/       房间图与奖励解析
  meta/      存档和条件叙事
  render/    Canvas 场景、WebGL 3D 机体、材质灯光、低成本敌机与兜底机体
  input/     键鼠与 iOS 多指触控
  ui/        基地、机体预览、路线、构筑、商店、结果界面
  data/      机体、模块、敌人、房间、对白数据
```

## 验证

```bash
npm run verify
```

验证包括语法检查、系统测试、移动布局回归、机体设计回归、WebGL 启动、Canvas 兜底、双摇杆并发输入、推进射击、镜头构图、战斗房流程、暂停恢复、存档、房间图、奖励槽位、枪口硬点和军刀扫掠测试。

真实浏览器 QA 使用 956×440、DPR 3、移动端触控环境，要求 Three.js 和 WebGL 实际启动，分别选择先锋、重装和高机动机体进入战斗并保存截图，同时检查 844×390 紧凑横屏布局。自动化环境使用 Chromium/SwiftShader，仍不能替代实体 iPhone Safari 的发热、功耗、触觉与长时间稳定性验证。
