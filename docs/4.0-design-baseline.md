# 《天穹断刃》4.0 设计基线

> 历史文档：保留 4.0 的系统推导，不描述 4.3 当前镜头、连续战线、移动布局或长期成长。当前事实以 `MECHA_MARCO_DESIGN_4.2.md`、代码和自动测试为准。

## 目标

停止继续向单文件原型堆叠功能。4.0 先重做产品骨架、战斗链路和代码模块，再扩充内容。

## 从《Hades》学习的核心

《Hades》真正成立依靠五条同时工作的系统链：

1. **瞬时战斗循环**：移动、普通攻击、特殊攻击、投射能力、闪避和终极能力各自具有明确用途与取消关系。
2. **单局构筑循环**：强化绑定动作槽位，并优先改变动作规则，而不只是提高数值。
3. **房间决策循环**：玩家进入房间前就能预判奖励，路线选择就是构筑选择。
4. **局外成长循环**：失败后仍获得永久资源、解锁、关系和长期目标。
5. **叙事反馈循环**：角色会读取玩家死因、到达区域、使用武器和事件标签，死亡本身也推进内容。

## 产品定位

固定斜俯视角的高速超级系机甲动作肉鸽。玩家驾驶原创机体，在一次次战役中重构武装与机体协议，并通过返航母舰推进角色关系、永久成长和世界事件。

## 设计支柱

- **看得懂**：敌我轮廓、攻击来源、危险区和奖励必须一眼可读。
- **动作对齐**：模型姿态、武器硬点、判定和特效由同一套数据驱动。
- **构筑改变动作**：优先改变射击方式、军刀动作、导弹逻辑和推进规则。
- **失败也推进**：每次返航都有资源、情报、对话或解锁。
- **短局完整**：移动版单次完整战役目标 20–30 分钟，普通房间 30–60 秒。

## 双循环

### 单局

```text
选择机体与武器
→ 查看两个奖励门牌
→ 战斗 / 精英 / 商店 / 维修 / 事件
→ 获取模块并改变动作
→ 中 Boss
→ 区域 Boss
→ 胜利或战败
```

### 局外

```text
返回母舰
→ 战役报告
→ 条件对话
→ 永久资源与解锁
→ 调整初始配置
→ 再次出击
```

## 三台机体

### 断刃·先锋型
- 主武器：光束步枪
- 特殊攻击：回旋军刀斩
- 推进：短冷却高速闪避
- 关键词：连射、折射、近远切换

### 断岳·重装型
- 主武器：爆裂加农炮
- 特殊攻击：盾击 / 蓄力重斩
- 推进：装甲冲撞
- 关键词：爆炸、护盾、反击、过热

### 星翼·高机动型
- 主武器：双联光束枪
- 特殊攻击：浮游刃突袭
- 推进：连续位移
- 关键词：锁定、多目标、残影、无人武器

## 动作槽位

每台机体统一拥有：

- `Primary`：主射击
- `Secondary`：近战或特殊武器
- `Dash`：推进
- `Ordnance`：导弹、浮游炮或重火力
- `Overdrive`：必杀或超限模式

强化必须声明作用槽位、标签、前置条件和规则变化。高级强化至少一半应改变动作规则，例如折射、回旋、召回、滞留、标记、爆炸和联动，而不是单纯增加伤害。

## 必须重做的技术系统

### Hardpoint Rig

武器模型、动画、攻击生成与特效共享同一套节点世界矩阵：

```text
Mech Root
├── Torso
│   ├── Right Shoulder → Forearm → Rifle → Muzzle
│   └── Left Shoulder → Saber Hilt → Saber Tip
└── Backpack → Missile L / Missile R / Thrusters
```

禁止再使用“玩家坐标 + 人工偏移值”生成武器。

### 动作状态机

```text
Idle / Move
DashStartup → DashActive → DashRecovery
PrimaryFire
SecondaryStartup → SecondaryActive → SecondaryRecovery
HitStun / Knockdown / Overdrive / Dead
```

每个状态声明持续时间、取消窗口、移动倍率、姿态、硬点动画、判定帧、特效帧和音效帧。

### 敌人 AI

```text
Spawn → Acquire → Approach / Strafe / Retreat
→ Telegraph → Attack → Recover
→ HitReact / Stagger / Enraged / Dead
```

受击反应根据攻击方向、伤害强度、霸体值和攻击类型计算后仰、侧倾、位移、停顿与硬直。

### VFX 分层

每次攻击至少包含：

- 发射层：枪口形状、光源、冲击环
- 弹道层：核心、外辉光、拖尾
- 命中层：定向闪光、冲击形状、装甲碎片
- 环境层：地面光、烟尘、残留痕迹

颗粒只作为辅层，不能替代主体形状。

## iOS 专项

移动版不是桌面版加虚拟按钮：

- 左摇杆移动。
- 右摇杆瞄准，超过阈值持续射击。
- 军刀、推进、导弹、必杀独立按钮。
- 支持辅助吸附，但不完全自动选目标。
- 动态降低同时敌人数，提高单体可读性。
- 危险区、字体和奖励卡针对横屏手机放大。
- 触控按下瞬间进入输入缓冲，不等待抬手。
- 中场菜单切换后清空所有触控状态。

## 验收优先级

```text
战斗动作对齐
> 操作手感
> 敌人和房间节奏
> 构筑差异
> 受击反馈
> 场景可读性
> 内容数量
```

## 关键验收标准

- 所有弹道从实际武器硬点生成。
- 军刀视觉轨迹、碰撞扫掠和机体动作来自同一节点序列。
- 每次命中都有方向性受击反馈。
- 玩家在 3 秒内理解敌方攻击预警。
- 每个动作槽位至少有 4 条构筑路线。
- 普通房间 30–60 秒，精英房 60–90 秒。
- iPhone 横屏无页面滚动、缩放和按钮卡住。
- 30 分钟自动压力测试无异常。
- 导弹目标死亡、清场、暂停、切后台和旋转屏幕都有回归测试。

## 实施顺序

1. **底层重构**：模块化源码、输入路由、状态机、Hardpoint Rig、命中和受击系统。
2. **一台机体做完整**：先锋型全部动作、一类敌人和一个完整房间。
3. **构筑和房间图**：奖励门牌、动作槽位强化、路径、商店、维修和事件。
4. **内容纵切**：三台机体、敌人、Boss、基地和局外成长。
5. **移动发布**：iOS 专项、性能分级、PWA 与 GitHub Pages。
