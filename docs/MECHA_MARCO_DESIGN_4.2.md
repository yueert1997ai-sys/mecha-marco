# 《天穹断刃 / Mecha Marco》4.2 当前设计基线

版本：`4.2.1-mobile-frontline-polish`

这份文档描述当前正式实现与下一阶段约束。历史推导见 `HADES_ARCHITECTURE_4.0.md`，实现冲突时以本文件、当前代码和自动测试为准。

## 1. 产品定位

纯俯视 `Top-down` 的机甲动作肉鸽，面向 iPhone 横屏和桌面网页。

玩家从开局就是危险的王牌机师，在 12 段连续战线中突破敌军基地、重构武装、恢复旧权限，并在返航基地后逐步发现自己与 `MA-00` 的关系。

核心体验：

- 单机突破整条防线
- 清场后继续驾驶推进，不频繁返回路线页
- 构筑改变动作和可见装备
- 失败、胜利与玩家行为共同推动基地剧情
- 爽感先行，身份悬念随后逐步发凉

## 2. 战斗视角与机体显示

### 2.1 镜头

- 战斗采用纯俯视投影
- 不使用斜俯视、三分之四战斗视角或平台透视倾斜
- 镜头保持战场边界稳定，只做有限跟随、冲刺响应和震动
- 世界坐标与瞄准坐标使用同一纯俯视映射

### 2.2 上半身俯视轮廓

正式战斗显示优先保留：

- 头部传感器
- 胸甲与核心
- 肩甲
- 双臂
- 右手主武器
- 左侧军刀结构
- 背包、翼板、导弹舱、护盾和浮游单元
- 推进器与尾焰

腿部在战斗姿态中隐藏。基地和改装预览可以展示更完整的机体结构。

机体可读性依赖装甲轮廓、武器方向、发光节点和挂载差异，不依赖高饱和色块。

## 3. 当前核心循环

```text
基地
→ 选择机体、涂装和操作设置
→ 出击轨道墓场
→ 连续战斗段
→ 闸门开启并保持控制推进
→ 关键分岔 / 奖励 / 商店 / 维修 / 事件
→ 第 12 段 Boss
→ 战役报告
→ 条件对白、人格恢复和档案更新
→ 再次出击
```

普通段落之间不返回路线页面。全屏 UI 只用于：

- 关键路线分岔
- 模块奖励
- 商店与维修
- 事件选择
- 暂停和结算

## 4. 轨道墓场 12 段结构

1. 封锁航道：建立单机突破爽感
2. 残骸雨带：移动危险和敌军异常识别
3. 断裂船坞：第一次关键分岔
4. 身份识别阵列：明确出现 `MA-00`
5. 封锁维修栈桥：补给与改装节奏
6. 英雄纪念舰列：暗示主角过去的英雄阶段
7. 封印输能带：确认区域承担封印功能
8. 监察官追猎：敌方短暂承认旧权限
9. 墓碑岔道：安全维护线与高风险旧档案库
10. 核心外环：文浩马开始主动覆盖上级命令
11. 守墓者前庭：公开广播封印对象正在接近
12. 轨道墓场核心：守墓者·阿尔法三阶段战

### 4.1 关键分岔

断裂船坞：

- 军械库航线：更强敌军和火力方向奖励
- 数据侧路：档案与舰队数据方向

墓碑岔道：

- 维护线：恢复耐久
- 旧档案库：高风险、精英增援、识别和档案恢复

分岔必须改变后续战役状态，不能只替换文案。

## 5. 当前战斗输入

桌面：

- 移动
- 瞄准与主射击
- 军刀
- 推进
- 挂载武器
- 超限
- 暂停

手机：

- 左摇杆移动
- 右侧主炮环按住射击并拖曳瞄准；关闭自动射击后，从主炮核心起按才会开火
- 推进和军刀位于右拇指高频区，挂载和超限位于外围
- 视觉按钮与触控热区分离，技能显示冷却、可用和超限蓄能状态
- 可调瞄准灵敏度、移动响应、瞄准死区、按钮透明度、自动射击和震动；只保留一套布局

输入按触点来源分别持有动作，允许移动、瞄准/射击、推进和军刀四指并发；旋转、后台恢复、pointer capture 丢失或滑出热区时清空失效触点。

## 6. 构筑结构

每台机体共享五个动作槽位：

- `Primary`
- `Secondary`
- `Dash`
- `Ordnance`
- `Overdrive`

构筑层包括：

- 基础数值模块
- 改变动作规则的核心改造
- 双体系组合协议
- 曙光 / 壁垒 / 蚀影三系共振
- 涂装和可见装备成长

商店或奖励安装模块后，战斗模型和基地改装预览必须同步变化。

## 7. 三台机体定位

### 断刃·先锋型

- 玩法：近远切换、稳定主射击、军刀联动、高频推进
- 视觉：锐利均衡、步枪与军刀清楚分离
- 叙事：战争英雄时期，代表荣誉、秩序和仍有底线的文浩马

### 断岳·重装型

- 玩法：爆裂重炮、护盾反击、重装冲撞和区域压制
- 视觉：宽肩厚胸、重武器、双反应炉背包
- 叙事：从王牌转向统治者的攻城机，真实用途涉及城市级防护设施

### 星翼·高机动型

- 玩法：多目标、连续位移、浮游武器和自动追踪
- 视觉：窄躯干、翼板、多推进节点和浮游单元
- 叙事：统治后期执行机，隐藏档案涉及失去武装后的持续追击

当前三台机体已经存在属性、动作倾向、外观挂载和涂装差异。下一阶段需要增加完全不同的专属资源机制。

## 8. MA-00 人格恢复

主角固定姓名：文浩马 / `WENHAO MA`

旧时代编号：`MA-00`

当前人格不是被另一个邪恶人格夺舍。恢复过程意味着记忆删除、身份重写和权限限制逐渐失效。

恢复进度读取：

- 出击、战败和通关
- Boss 见面与击杀
- 使用机体
- 主射击、军刀和挂载击杀倾向
- 低血清场
- 高风险路线
- 投降接受或拆解
- 档案碎片

人格阶段：

```text
服从任务
→ 旧习惯回归
→ 主动决定目标
→ 覆盖上级命令
→ 恢复指挥权限
→ 不再承认任何人拥有最终约束权
```

第一章只推进到“开始夺回决定权”。

## 9. 基地与叙事循环

基地承担：

- 机体和涂装选择
- 操作设置
- 战役结算
- 条件对白
- 双层机体档案
- 人格恢复和指挥权限反馈
- 下一次出击

主要角色：

- 舰长·零：上级逐渐变成确认者和服从者
- 整备长·伊芙：维护机体、人格同步、旧关系证人和可能的共犯
- 战术官·洛岚：教学、战术分析和旧人格测试
- 守墓者·阿尔法：第一个明显认出文浩马的 Boss

对白必须成熟、简短、冷静、有军事感，不使用热血少年喊招或长篇解释。

## 11. 已知边界

- 多数段落仍以清敌为主；身份识别阵列、封印输能带和核心外环已加入可摧毁设施目标
- 12 段是连续推进的分段战场，不是无缝开放世界
- 永久成长树和角色关系系统仍较浅
- 程序化机甲尚未替换为正式 GLB 资产
- Three.js 使用固定 CDN，弱网时依赖 Canvas 兜底
- 自动测试使用 Chromium / SwiftShader，不能代表实体 iPhone Safari
- 手柄、正式音频许可、实体障碍导航和复杂 Boss 机制尚未完成

## 11.1 4.2.1 场景空间规则

- 十二段分别使用封锁通道、残骸岛、船坞走廊、扫描航道、防守接驳区、纪念环、输能环、追击线、墓碑迷宫、移动闸门、前庭和 Boss 核心空间
- 每段拥有独立地面图形语言，并只绘制当前与相邻航段
- 程序化障碍参与玩家和敌人碰撞；敌人碰撞后获得侧向脱困速度，避免持续顶墙
- 设施目标拥有耐久、受击、摧毁反馈和 HUD 剩余数量，清敌但未摧毁设施时不会开门
