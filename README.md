# Mecha Marco / 天穹断刃

面向 iPhone Safari 与桌面浏览器的纯网页机甲动作肉鸽。

- 正式版本：`4.4.0-vanguard-identity`

- 当前开发基线：`v4/`
- 历史兼容版本：`v3/`

- 战斗机体轮廓：以上半身、肩甲、武器、背包和推进结构为主要可读信息；腿部在正式俯视战斗轮廓中隐藏

## 4.4 当前核心流程

```text
母舰基地
→ 选择机体、涂装与设置
→ 进入轨道墓场
→ 12 段连续战线
→ 清场后开启物理闸门并继续驾驶推进
→ 关键分岔 / 战场奖励 / 商店 / 维修 / 事件
→ 三阶段守墓者·阿尔法
→ 战役报告
→ 基地条件对白、档案与人格恢复
→ 再次出击
```

4.3 在 4.2 连续推进基础上补齐任务状态机与长线闭环，不再是每关返回路线页面。十二段分别承担突破、残骸规避、占领、设施破坏、防守、序列占领、限时过载、追击、伏击、闸门破坏、指挥编队和阶段 Boss；任务成败与路线选择会通过战术回执明确说明，并持续改变后续敌群、设施、补给、维修与 Boss 支援。

4.4.0 在不扩展其他机体或关卡的前提下，为先锋机与 OG-04 增加首个机体身份纵切：主炮/军刀命中积累刃势，短蓄军刀偏转敌方投射物或军刀，满足 35 刃势后将下一发主炮转化为高贯穿反击射击；OG-04 同步加入可读的巨构遗迹、四态识别塔、真实障碍和开关闸门。

## 当前战斗与操作

- 纯俯视战斗镜头，不再使用斜俯视或三分之四战斗视角
- 玩家、精英与 Boss 使用 WebGL 机甲显示层；普通敌人使用低成本 3D 结构
- 玩家机体以俯视上半身轮廓呈现，强化肩甲、头部、胸甲、武器、背包、挂舱与推进器识别
- 键鼠操作支持移动、瞄准、主射击、军刀、推进、挂载和超限
- iPhone 横屏使用左摇杆移动、右侧主炮环按住射击并拖曳瞄准，军刀、推进、挂载和超限按频率分层排布
- 设置支持瞄准灵敏度、移动响应、瞄准死区、按钮透明度、震动和推杆自动射击；布局保持统一
- iPhone 安全区、Home Indicator、短横屏布局与多指触控已纳入自动回归

## 当前构筑

- 三台差异化机体：断刃·先锋型、断岳·重装型、星翼·高机动型
- 五个动作槽位：`Primary`、`Secondary`、`Dash`、`Ordnance`、`Overdrive`
- 基础模块、稀有规则改造、组合协议和三系共振
- 部分强化会改变射击、军刀、推进、挂载和超限的动作规则
- 商店或奖励安装模块后，战斗模型与基地改装预览会同步改变挂载、肩甲、翼片、护盾、发光通道和浮游单元
- 三台机体各有多套涂装，战斗与基地预览同步

## 当前区域与剧情

第一区域“轨道墓场”由 12 段连续战线组成，包含两次关键路线分岔、战场奖励、补给、事件和 Boss。

底层叙事围绕文浩马 / `WENHAO MA` / `MA-00` 展开：

- 战败后母舰回收驾驶数据并使用旧档案修复人格缺失
- 出击次数、战败、Boss、机体、武器倾向、高风险路线、低血通关、投降与残骸选择共同影响恢复进度
- 基地对白读取战役标签，不按固定顺序播放
- 战役报告逐步混入人格匹配率、指挥权限和旧档案字段
- 三台机体拥有公开档案与封存档案两层信息
- 第一章只确认敌方认识文浩马、母舰隐瞒身份、MA-00 权限正在恢复，不提前揭露全部真相

## 基地循环

基地当前承担：

- 机体与涂装选择
- 手机操作设置
- 战役结果与永久资源
- 条件对白和角色关系反馈
- 机体公开 / 封存档案
- 下一次出击入口

永久成长采用不叠加裸属性的横向军备体系：每台机体有三套改变开局动作倾向的战术套件，侧向套件分别需要 8 / 24 点对应机体熟练认证后再消耗舰队数据签发；通关逐步解锁风险指令；十二份战线档案直接显示可执行挑战。风险指令最多同时启用三条，用更高敌压、禁修、强化危险区或 Boss 狂暴换取更高回收倍率。

## 已知边界

4.4.0 是第一章的先锋机身份纵切，还未达到完整发行版本。

- 护送与可失败后的替代结算尚未形成完整目标族；当前防守失败仍以本局惩罚为主
- 12 段已经连续推进，但仍是分段战场加载，不是无缝开放世界
- 三台机体已有动作、属性、涂装和外观差异，专属资源机制仍需深化
- MA-00 叙事底层已接入，角色关系、分支事件和后续区域内容仍需扩充
- 程序化机甲已可用，仍未替换为正式美术流程导出的 GLB 资产
- Three.js 继续从固定 CDN 加载；离线或网络异常时使用 Canvas 兜底
- 自动 QA 使用 Chromium 与 SwiftShader，不能等同于实体 iPhone Safari / WebKit GPU
- OG-04 自定义视觉层仍会覆盖一次已经执行的通用场地绘制，属于当前非阻塞性能成本
- 尚未完成实体 iPhone 17 Pro Max 的 30 分钟持续性能、发热、功耗、内存与真实多指压力测试
- 手柄输入、正式音频资产与许可清单、复杂群体寻路和更多 Boss 招式仍待完善
